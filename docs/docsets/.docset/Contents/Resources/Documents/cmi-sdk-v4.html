CMi SDK v4

Fourth generation of CASE Mobile integration SDK with revised inter-app communication based on app links and universal links.

CMi SDK consist of 3 components:

  • CMi Core - core module - contains common classes, constants used by other modules
  • CMi TP - third party module - used by TPA to create requests and processing responses for supported scenarios
  • CMi CM - case mobile module - used by Case Mobile application to process requests from TPA, communicate with MEP and return responses to TPA

Glossary

Description of term and abbreviations used in documentation (ordered alphabetically):

  • A/U link: app link (for Android) / universal link (for iOS)
  • CASE: Authentication system, supporting various methods of - authentication & authorization.
  • CASE mobile: CASE is supporting authentication of user & authorization of transactions using smartphone. Term ‘CASE mobile’ is used to refer mobile application that is able to communicate with CASE.
  • CMi: CASE mobile integration.
  • CMi Core: CMi SDK module with common functionality
  • CMi TP: CMi SDK module for TPA
  • CMi CM: CMi SDK module for CM
  • CM (application): CASE mobile
  • MEP: Multichannel entry point
  • OS: Operating system (Android, iOS, Windows Phone)
  • SDK: software development kit, library
  • TPA: Third party application, that is using CMi TP SDK for - communication with CASE mobile

General description

A/U links have slightly different behavior on iOS and Android. On iOS, request made by opening URL (that applications in listening to) causes OS to pass control to application by calling method in AppDelegate. Android can declare in manifest, which application component should be invoked by opening particular URL.

CMi SDK provides API for processing URL requests, but it does not restrict nor force (Android) application to use particular component to receive URL requests from OS.

However, CMi SDK will use URLs with particular form and applications should use it in configuration of A/U links.

To setup A/U links properly, refer to official documentation:

  • Android tutorial
  • iOS docs
